Features

Everything your desktop needs

A tiny AI companion that lives on your desktop, learns about you, and has a personality of its own.

⚙

Local AI (Ollama)

Powered by llama3.2:1b running entirely on your machine via Ollama. No cloud, no accounts, no data leaks.

☾

Neon Robot

A custom-painted animated robot face with idle, thinking, sleepy, happy, and curious states.

♡

Persistent Memory

Teach it facts about you. Say "remember that" or "I like" and it stores preferences locally.

☺

Idle Personality

Gets curious, sleepy, or excited on its own. Click it to toggle between mini and expanded modes.

☰

Settings

Customize theme, colors, timers, memory capacity, and thinking depth through a built-in settings panel.

☘

Linux & macOS

Runs on any system with Python 3. Install via pip or your native package manager and go.

Installation

Up and running in minutes

Choose your platform and follow the commands. All you need is Python 3 and Ollama.

# Clone the repo git clone https://github.com/Chenboda01/ScreenBot.git cd ScreenBot # Install dependencies (Arch Linux) sudo pacman -S python-pyqt6 python-requests # Pull the AI model ollama pull llama3.2:1b # Launch the bot python3 ScreenBot.py
# Clone the repo git clone https://github.com/Chenboda01/ScreenBot.git cd ScreenBot # Create and activate a virtual environment python3 -m venv .venv source .venv/bin/activate # Install dependencies python3 -m pip install PyQt6 requests # Download Ollama from ollama.com, then: ollama pull llama3.2:1b # Launch the bot python3 ScreenBot.py
# Install Waydroid sudo pacman -S waydroid # Initialize Waydroid sudo waydroid init # Start Waydroid sudo systemctl start waydroid-container waydroid session start # Launch Android UI waydroid show-full-ui # Inside Android, install Python pkg install python # Copy ScreenBot into Waydroid, then: pip install PyQt6 requests
Linux Host │ ├── Ollama │ └── Waydroid (Android Emulator) │ └── ScreenBot

Ollama does not run inside Android. ScreenBot in Waydroid connects to Ollama on the Linux host using http://HOST_IP:11434 instead of localhost.

# Install Android Studio # Download from developer.android.com/studio # Create a virtual Android device # via Android Studio's AVD Manager # Install Python inside the Android emulator # Copy ScreenBot into the emulator # Install dependencies inside Android pip install PyQt6 requests # Run Ollama on the Windows host ollama pull llama3.2:1b # ScreenBot connects to host Ollama via IP
Windows Host │ ├── Ollama │ └── Android Emulator (AVD) │ └── ScreenBot

Use the Android Emulator included with Android Studio. ScreenBot connects to Ollama running on the Windows host.

# Install BlueStacks # Download from bluestacks.com # Install Python inside BlueStacks # Copy ScreenBot into BlueStacks # Install dependencies inside BlueStacks pip install PyQt6 requests # Run Ollama on your host PC ollama pull llama3.2:1b # ScreenBot connects to host Ollama via IP
Desktop Computer │ ├── Ollama │ └── BlueStacks │ └── ScreenBot

BlueStacks works similarly to other Android emulators. The AI stays on your computer — the emulator only displays the robot.

How to Use

Your new companion

Once running, ScreenBot lives on your desktop. Here's how to get the most out of it.

1

Run the app

Launch with python3 ScreenBot.py. The robot appears in a small window (180x160) on your desktop. Ollama must be running with llama3.2:1b pulled.

2

Click to expand

Click the robot to toggle between mini mode (180x160) and expanded mode (520x670) with the full chat interface and settings.

3

Teach it about you

Use natural phrases in chat: remember that I'm a developer, I like dark mode, I prefer tea over coffee. ScreenBot stores these locally in ~/.screenbot_memory.json.

4

Customize settings

In expanded mode, open Settings to change the theme, text color, background color, curiosity timeout, sleep timer, memory capacity, and thinking depth.

Architecture

How ScreenBot fits together

ScreenBot runs inside an Android emulator while the AI engine stays on your desktop.

Desktop Computer │ ├── Ollama — Local AI engine │ ├── Git — Version control │ └── Android Emulator │ └── ScreenBot — The robot companion

The AI stays on your computer. The Android emulator only displays the robot. ScreenBot inside the emulator connects to Ollama on the host machine via the host's IP address.

Current Limitations

What ScreenBot can't do yet

ScreenBot is a desktop companion, not a mobile app. Here's what to keep in mind.

  • ✗ Not designed for Android phones
  • ✗ Not designed for tablets
  • ✗ PyQt6 is not officially supported on Android
  • ✗ The robot may require a different GUI toolkit in the future
Future

Where ScreenBot is heading

Two separate applications — one for desktop, one for mobile.

ScreenBot Desktop

The desktop version will continue using PyQt6, providing the full animated robot experience with native performance on Linux, macOS, and Windows.

ScreenBot Mobile

The mobile version will be rewritten using a mobile framework — such as Qt for Android or another suitable toolkit — rather than trying to run the desktop code unchanged.

Eventually these will be two separate applications, each optimized for their platform's strengths and user expectations.
